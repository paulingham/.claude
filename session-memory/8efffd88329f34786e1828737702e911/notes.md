# Patterns & Conventions
_Code patterns in use (service objects, barrel exports, hook composition). Naming conventions. Architecture decisions observed. Framework-specific idioms._

- Three-tier progressive disclosure: search → timeline → get_observations / get_findings.
- `source` parameter dispatch: `observations | scratchpad | both` across all three tiers (keeps S4 MCP schema symmetric).
- Clean-room port — no thedotmack/claude-mem code (AGPL-3.0; `ragtime/` PolyForm-NC).
- JSON envelope `{tier, hits, total, truncated}` — MCP-ready for S4.
- argparse `choices` for source validation; fixed filter whitelist per source; `?` placeholders only (no f-string interpolation of user data into SQL).
- Shape: bodies ≤5 lines, files ≤50 lines, CC ≤5, nesting ≤2.
- Byte-budget ACs tested with realistic fixtures (≥150 char bodies, full content_hash, populated secondary sources).
- **Facade routing** for optional/swappable backends (embedder): single `get_X()` function, never direct imports. Preserves test seam and swap point.
- **Shape-driven decomposition** at 50-line limit: `_lib/{module}_actions.py` (handlers) + `_lib/{module}_batch.py` (batch/write) + `_lib/{module}_cli.py` (argparse + formatter). Keeps public module tiny (dispatch dict + 5-line main).
- **Lazy imports** MANDATORY for gated-import invariants (capture zero-cost). Function-local `from X import Y`, not top-level.
- **Honest docs**: user-facing SKILL.md/CLI examples/download scripts must reflect actual ship state. Happy-path examples that imply unimplemented functionality = quality footgun (caught by product-reviewer; cost an accept round).
- Read-only invariant proven 3 ways: behavioural test + static grep + PRAGMA post-check.
- **`_ensure(path, step)` idiom** (S9): tiny wrapper that runs `step()` only if `path` doesn't exist. Keeps orchestration declarative ("ensure this exists, and if not, run this step"). Reusable for install/probe patterns.
- **Generic + specific settings patcher split** (S9): `settings_patch.py` is domain-agnostic (patches any JSON env key atomically via mkstemp + os.replace); `bootstrap_settings.py` is the embedder-specific wrapper (dylib guard + try/catch). Separation of concerns; `settings_patch` is reusable for future env-key patches.
- **`_run_timed` helper** (S9): every `subprocess.run(..., timeout=N)` must catch `subprocess.TimeoutExpired` — timeout alone causes raise, not return. When 2+ call sites share the shape, extract a helper.
- **Never-raise boundary** (S9): bootstrap `run()` is the contract boundary — every internal failure path (brew absent, brew non-zero, download non-zero, JSON parse error, settings write error, subprocess timeout) must log WARN and return a partial code. The boundary helper aggregates individual partial codes via OR.
- **Subprocess byte-identity regression guard** (s9-residual, `tests/test_bootstrap_settings_disk_invariant.py`): snapshot `read_bytes()` before, run suspected-leaky suite in subprocess with `env=` controlling config path, compare `read_bytes()` after. Cheap, falsifiable, no stubbing of SUT. Dual-snapshot (tempfile + real settings file) = defence-in-depth against pollution vectors.
- **PYTHONPATH split by cwd invariance** (s9-residual): relative `skills` when command runs from `~/.claude`; absolute `$HOME/.claude/skills` when command runs from user's arbitrary project root (e.g. `/project-setup`). Do NOT conflate — always ask "where is this command invoked from?"
- **Env-backed path tests need tempdir overrides** (s9-residual): tests that exercise code writing env-backed paths (ORT_DYLIB_PATH, CLAUDE_SETTINGS_PATH) must override `CLAUDE_SETTINGS_PATH` to a tempdir via `patch.dict(clear=False)` — otherwise they leak into real `~/.claude/settings.json`.
- **In-PR scope amendment over defer-and-file** (s9-residual): 4-slice PR retained minimum-diff posture via two in-PR amendments (M1 surfaced in build re `python3 -m embedder` broken invocations; M2 surfaced in review). User principle: "docs directing users to broken commands is itself a defect" — fold the fix into the current PR rather than shipping broken docs with a follow-up ticket. See `feedback_in_build_discovery_expands_scope.md`.
- **Serial vs parallel merge strategy** (monolith-first): slices touching the same high-churn file (e.g. `CLAUDE.md`) merge SERIALLY; slices touching disjoint files merge in PARALLEL. Cleanly avoided merge conflicts across 6 slices. Apply this split on any multi-slice pipeline where one file is touched by >1 slice.
- **In-cycle fix, same-reviewer re-review** (monolith-first): two CHANGES_REQUESTED recovery loops (Review + QA) both resolved by spawning fix-engineer in worktree, merging, then re-reviewing with the SAME reviewer/analyst with scope narrowed to just the findings. Preserves reviewer context, enforces no-follow-up discipline.
- **AC-as-test for docs-only pipelines** (monolith-first): grep-based FF SSOT discipline check, FF keyword parity across 3 skill files, skill-registry smoke test, cross-reference integrity. Tier 3 (mutation) and Tier 4 (E2E) explicitly marked N/A with rationale — docs-only changes don't have behavior to mutate.
- **OS detection idiom** (cloud-portability): `awk -F= '$1=="ID"' /etc/os-release` with quote-stripping. Path env-overridable via `OS_RELEASE_PATH`.
- **Installer tri-state** (cloud-portability): `--dry-run` prints, no flag prints+exits 1, `--yes` executes. Never implicitly mutates.
- **Idempotent symlink**: `ln -sfn`, never `ln -s`.
- **Shell DIP via printer env vars** (cross-env): `INSTALL_PKG_CMD_PRINTER`, `CLAUDE_RUST_PRINTER` — inject echo-like side-effect endpoints for hermetic tests. Idiomatic bash DI; no classes needed.
- **Tri-state env var for test seams** (`1|0|auto`): preserves production behaviour AND allows forcing either branch. Two-valued flags underserve one of three cases.
- **Test hook on the unit, not the collaborator**: `CLAUDE_RUST_FORCE_DISTRO_FAIL=1` lives inside `_rust_try_distro` rather than monkey-patching `install_pkg`. Keeps hermetic contract scoped to unit under test.
- **RED injection for regression guards**: when a guard is already satisfied, temporarily inject an offender to observe RED, then restore. Example: M2 `declare -A` injection to confirm bash-compat.bats fires.
- **awk-scope assertions**: structural assertion that prose only appears inside a specific case-arm block. Stronger than grep substring match.

- **Harness hook removal checklist** (PR #21): when removing a harness hook always delete (a) the script itself in `hooks/`, (b) its entry in `settings.json` via jq safe-prune (see Critical Paths), (c) its test block in `hooks/tests/test-hooks.sh`, (d) any orphan runtime state files in `~/.claude/automation/`. Document the emergency edit in pipeline state.

# Agent Effectiveness
_What approaches worked well. What wasted time. Which agent types were most effective for which tasks. Optimal configurations for this project._

- `SendMessage` to an existing write-capable agent sidesteps the worktree-isolation hook block on round-N fixes (agent already owns a worktree). Faster than spawning fresh.
- `product-reviewer` is read-only; orchestrator must persist its verdict state files (reviewers cannot Write).
- Max-2-round review rule + explicit QA gap-analysis round catches what reviewers structurally miss (byte budgets, cross-source ordering).
- Read-only reviewers (code-reviewer, security-engineer) MUST be spawned as persistent team teammates so they remember context across rounds.
- Architect → interactive plan approval is fast when user is hands-on; challenger team (product-reviewer + software-engineer) only needed in autonomous mode (per `feedback_plan_validation_autonomous.md`).
- S5 pattern validated: Plan Validation autonomous round 1 challenger CHANGES_REQUESTED → architect revises → round 2 APPROVE. 2-round cap held. Challengers remember plan context.
- S5 pattern validated: Accept-phase retrofit (not rebuild) is viable when infrastructure passes verify/test but user-facing surface fails. Scope of fix: 30-60 min, focused diff, targeted re-accept on the failing ACs only.
- Write-capable agents blocked without `isolation: "worktree"` — session memory updates must route through orchestrator Edit (memory is in the config-edit exception scope).
- General-purpose and Explore agents blocked at hook level; use software-engineer with worktree, or orchestrator-direct for memory/pipeline-state work.
- **cross-env batch: parallel slice build** (slices 1-4 in parallel, slice 5 serial) was the right call — only slice 3 touched all files, so serialising it last would have conflicted with everything. Parallel 1-4 + slice 5 after merge kept conflict surface at one orchestrator-delegated merge (settings.json).
- **cross-env batch: Sonnet sufficient for slice 5 build** (polish scope, no architectural decisions). Opus warranted for slices 1-4 (three-way OS split, bootstrap consent gate, PPID scoping, CRITICAL source-fallback fix).
- **Standalone-subagent reviewers > team-dispatched reviewers** for this harness right now. Investigate team-dispatch idle failure before reverting.
- **opus47-thinking**: in-process teammates can become unresponsive across long pipelines — `SendMessage` queue stalls observed. Workaround: fresh subagent spawn for re-review, accept lost context, verify fix-engineer commit on its own.
