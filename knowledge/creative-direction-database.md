# Creative Direction Database

## Banned Defaults (Anti-Convergence)

These fonts and colors are the statistical center of AI design output. Use them ONLY if explicitly requested by a brand brief:

```
BANNED FONTS: Inter, Roboto, Open Sans, Arial, Helvetica, Lato, Montserrat
BANNED PALETTES: Default blue (hsl 210-230), Bootstrap blue, Material blue
BANNED LAYOUTS: Centered hero with gradient background + 3-column card grid below
BANNED PATTERNS: Top nav bar with logo-left/links-right on mobile (use bottom sheet)
```

If your first instinct produces any of these → STOP → pick from the curated alternatives below.

### Palette Generation Formulas
```
Monochromatic:        Single hue, vary lightness 15%-85%, saturation 30%-80%
Analogous:            Base hue ± 30°, same lightness range
Split-complementary:  Base hue + (180° ± 30°) — vibrant but balanced
Triadic:              Base hue + 120°, + 240° — for dashboards with many data series
```

## Layout Archetypes

### Asymmetric Bento Grid
```
Large feature card spanning 2 columns + 2 rows
Smaller cards filling remaining grid cells asymmetrically
Variable cell sizes create visual hierarchy through scale
Gaps: 12-16px. Cards have distinct content types (stat, chart, list, image)
Best for: Dashboards, portfolios, feature showcases
```

### Magazine Editorial
```
Large hero image (full-bleed or 70% width)
Pull quotes in contrasting type
Multi-column text flow (2-3 columns on desktop)
Asymmetric image placement breaking the grid
Generous whitespace between sections (64-96px)
Best for: Content-heavy, storytelling, brand sites
```

### Dashboard Density
```
Compact spacing (8-12px gaps), small type (13-14px body)
Information-dense without clutter
Collapsible sidebar navigation
Data table as primary content with inline actions
Monospace numbers for alignment, tabular figures
Best for: Admin panels, analytics, monitoring
```

### Generous White Space
```
Oversized type scale (body 18-20px)
Single focal element per viewport
Spacing between sections: 96-128px
Content width maxed at 680px for readability
Photography or illustration as breathing space
Best for: Luxury brands, premium products, portfolios
```

### Card Mosaic (Masonry)
```
Variable-height cards in a masonry/waterfall layout
Auto-sizing based on content
Load-more or infinite scroll
Hover reveals action overlay
Best for: Image galleries, social feeds, discovery
```

### Vertical Feed (Social-Influenced)
```
Full-width content cards in vertical scroll
Swipe-to-action (dismiss, save, share)
Bottom sheet navigation (no top nav bar on mobile)
Thumb-zone optimized controls (lower 40% of screen)
Immersive: content fills viewport, controls overlay
Best for: Content apps, social, news, discovery
```

## Visual Texture Patterns

### Gradient Meshes
```css
background: radial-gradient(at 20% 30%, hsl(var(--primary) / 0.3) 0%, transparent 50%),
            radial-gradient(at 80% 70%, hsl(var(--accent) / 0.2) 0%, transparent 50%),
            hsl(var(--background));
Use: hero sections, page backgrounds, card highlights
```

### Grain / Noise Overlays
```css
/* Adds warmth and tactility */
background-image: url("data:image/svg+xml,..."); /* tiny noise SVG */
/* Or CSS-only: */
filter: url(#noise); /* SVG filter reference */
opacity: 0.03-0.08; /* subtle, not overwhelming */
Use: backgrounds, card surfaces, image overlays
```

### Glassmorphism 2.0
```css
/* Dark base with frosted panels */
.glass-panel {
  background: hsl(var(--foreground) / 0.05);
  backdrop-filter: blur(12px) saturate(1.5);
  border: 1px solid hsl(var(--foreground) / 0.08);
  border-radius: var(--radius-lg);
}
Use: cards on dark backgrounds, modals, overlays
```

### Perspective Transforms
```css
.card-perspective {
  transform: perspective(1000px) rotateY(-2deg);
  transition: transform 0.3s cubic-bezier(0.4, 0, 0.2, 1);
}
.card-perspective:hover { transform: perspective(1000px) rotateY(0); }
Use: feature showcases, interactive card grids (sparingly)
```

### Clip-Path Sections
```css
.angled-section {
  clip-path: polygon(0 0, 100% 5%, 100% 100%, 0 95%);
}
Use: section dividers, hero transitions, visual breaks
```

### Animated Gradient Borders
```css
.gradient-border {
  background: linear-gradient(var(--background), var(--background)) padding-box,
              linear-gradient(135deg, var(--primary), var(--accent)) border-box;
  border: 2px solid transparent;
}
Use: featured cards, active states, premium elements
```

## Design Philosophy Modes

### Simplicity Through Reduction
Remove elements until something breaks. The minimum effective design.
- Every element must justify its existence
- If removing it doesn't hurt comprehension → remove it
- White space is a feature, not waste

### Material Honesty
UI elements look and behave like what they are.
- Buttons look pressable (shadow, depth)
- Inputs look editable (visible border, clear affordance)
- Links look followable (color, underline)
- No fake 3D that doesn't respond to interaction

### Functional Layering
Each visual layer serves a specific communication purpose.
- Layer 1: Background (establishes mood)
- Layer 2: Content containers (groups related info)
- Layer 3: Interactive elements (calls to action)
- Layer 4: Overlays (modals, tooltips — temporary focus)

### Obsessive Detail
Every pixel is intentional. Optical alignment over mathematical.
- Text baseline alignment across columns
- Icon optical centering (not bounding-box center)
- Consistent icon stroke width across the set
- Sub-pixel rendering awareness for thin lines

### Future-Forward
Design for how users interact with TikTok, Instagram, and voice assistants — not how they used desktop apps in 2020.
- Gesture-rich: swipe, pull, drag as primary navigation
- Voice-ready: every flow has a voice-input path
- Content-first: UI chrome dissolves, content fills viewport
- Ambient intelligence: interface adapts without explicit user configuration
- Social-native: share, save, react as first-class actions
- See `next-gen-interaction-patterns.md` for full implementation patterns
