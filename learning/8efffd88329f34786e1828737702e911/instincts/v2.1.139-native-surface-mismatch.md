## Pattern

v2.1.139 shipped `continueOnBlock`, `x-claude-code-agent-id`/`x-claude-code-parent-agent-id`,
and `autoMode.hard_deny`. The names suggest they replace Path-B advisory hooks,
agent-frontmatter parent walking, and the destructive-verbs.txt guard respectively.
They do NOT.

- `continueOnBlock` is a PostToolUse-only field — fires after the tool ran. Useless for
  pre-spawn validation (allowlist, thinking, depth, runtime caps). The Path-B disclosures
  in `rules/_detail/thinking-defaults.md` and `rules/_detail/agent-protocol.md` § Per-Agent
  Tool Scoping describe the actual missing surfaces (`thinking`, `allowed_tools` on Agent
  input schema). Don't conflate.

- `x-claude-code-agent-id` is an outbound API request header (and OTEL span attribute) for
  identifying agent instances to the Anthropic backend. It is not visible to local hooks,
  and it identifies runtime instances, not frontmatter ancestry. `agent_parent_chain.py`
  walks `parent:` frontmatter declarations across `agents/*.md`; the headers don't carry
  that information.

- `autoMode.hard_deny` is a prose-rule classifier that fires ONLY in auto mode. The existing
  `hooks/main-branch-guard.sh` destructive-verb path fires in every mode. hard_deny is at
  best additive (belt-and-braces for auto-mode users), never a replacement.

When evaluating a future v2.1.X feature for migration potential: read the canonical hooks
and settings reference pages (NOT the changelog one-liner) and confirm WHICH HOOK EVENT the
feature lives on and WHICH SESSION MODES it applies to. Many features named to suggest
broad enforcement are scoped to specific modes or specific lifecycle phases.
